# TelegramUserBot

## Введение

TelegramUserBot — это мощный Telegram-бот на Java с использованием библиотеки TelegramBots. Бот идеально подходит для управления базой VPN-пользователей и автоматической рассылки.

**Основные возможности:**
- 💾 Сохранение базы пользователей в `users.json` без потерь при перезагрузке.
- 🎛️ Интерактивная панель администратора на базе Inline-клавиатур с подменю, удобной системой навигации (кнопки «Назад») и защитой от засорения чата (используется `EditMessageText`).
- 📬 Асинхронные рассылки (текст и медиа) с исключением забаненных.
- 🚫 Система банов с автоматическими уведомлениями пользователям.
- ⚡ Быстрая отправка ЛС: напишите `<Имя> <Сообщение>` прямо в чат без нажатия кнопок.
- 🔑 **Продвинутая интеграция с 3x-ui**:
  - Автоматические напоминания об истечении VPN-подписок.
  - Синхронизация и бекап базы панели в пару кликов.
  - Пользователи могут запросить свои прямые ключи (VLESS, VMess, Hysteria и др.) и ссылку на подписку прямо в боте.
  - Администраторы могут просматривать статистику по конкретному юзеру и сбрасывать ему трафик в панели.

## Настройка и запуск

### 1. Конфигурация (`config.properties`)

Создайте или откройте файл `src/main/resources/config.properties`:

```properties
# Telegram Bot Configuration
bot.token=ВАШ_ТОКЕН_БОТА
owner.id=ВАШ_ID_ВЛАДЕЛЬЦА
bot.username=@ИМЯ_ВАШЕГО_БОТА

# 3x-ui Panel Configuration
xui.panel.url=https://your-domain.com:54321/custom-path
xui.panel.username=admin
xui.panel.password=your_password
xui.api.token=ТОКЕН_ИЗ_НАСТРОЕК_БЕЗОПАСНОСТИ_ПАНЕЛИ
xui.sub.port=*port*
xui.sub.path=/sub/your-custom-sub-path/

# Как часто проверять срок ключей (в часах)
xui.check.interval.hours=6

# За сколько дней до конца отправлять уведомление (через запятую)
xui.notify.days=3,1,0

# Фильтр по инбаунду: только клиенты с этого порта (закомментируйте для всех)
# xui.inbound.port=443
```

> `owner.id` — ваш Telegram user ID (узнать можно через [@userinfobot](https://t.me/userinfobot)).

### 2. Сборка и запуск

```bash
mvn clean package
java -cp target/telegram-userbot-1.0-SNAPSHOT-jar-with-dependencies.jar com.example.TelegramUserBot
```
В Linux рекомендуется запускать как systemd-сервис.

## Возможности Администратора

Панель администратора удобно разбита на категории:

### 👥 Пользователи
- **📋 Список**: Реестр всех зарегистрированных.
- **ℹ️ Информация**: Детальная сводка по конкретному юзеру (Telegram ID, XUI ID, даты окончания подписки, статус в сети, потребление ресурсов). Позволяет сбросить ему трафик.
- **✏️ Переименовать**, **🗑️ Удалить**, **🚫 Забанить / ✅ Разбанить**, **📊 Статистика**.

### 📢 Рассылка
- **✉️ Сообщение юзеру / 📤 Медиа юзеру**: Отправить личное сообщение / файл.
- **📢 Текст всем / 🖼️ Медиа всем**: Асинхронные массовые рассылки сразу всей активной аудитории бота.

### ⚙️ Панель 3x-ui
- **🔄 Синхронизация**: Принудительно сопоставить юзеров из бота с базой 3x-ui (по Email) и обновить базу истечений.
- **💾 Бэкап**: Выгрузить файл резервной копии базы `.db` вашей 3x-ui напрямую в Telegram.

## Функции Пользователя

При обращении к боту, юзер увидит:
- Привязка Telegram-аккаунта (при первом входе) к его 3x-ui email.
- **🔑 Мои ключи**: Позволяет моментально запросить все свои активные конфигурации VPN, а также единую подписочную ссылку из панели (скачивается напрямую через Sub URL, игнорируя HTML темы панели для чистоты парсинга).
- **📝 Написать админу**: Обратная связь, пересылающаяся владельцу.

## Интеграция с 3x-ui

Бот автоматически подключается к вашей 3x-ui панели (используя API Token) и отслеживает сроки действия подписок во всех настроенных "Inbounds".

**Как работает удержание клиентов:**
1. Каждые N часов бот берет список клиентов со всех инбаундов `settings.clients`.
2. Находит совпадение email клиента с базой бота Telegram.
3. Если до истечения ключа осталось указанное число дней — отправляет уведомление:
   - За *3 дня*: напоминание, что срок истекает.
   - За *1 день*: срочное напоминание о возможности блокировки.
   - За *0 дней (в день истечения)*: финальное предупреждение.
> Имя пользователя в боте во время регистрации должно совпадать с Email (названием профиля клиента) в вашей 3x-ui панели.

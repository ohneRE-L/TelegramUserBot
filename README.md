# TelegramUserBot

## Введение

TelegramUserBot — это мощный Telegram-бот на Java с использованием библиотеки TelegramBots. Бот идеально подходит для автоматизации взаимодействия с пользователями и проведения массовых рассылок.

**Основные возможности:**
- 💾 Автоматическое сохранение базы данных в локальный файл `users.json` без риска потерь при перезагрузке.
- 🎛️ Графическая панель администратора внутри Telegram на базе Inline- и Reply-клавиатур.
- 📬 Стабильная рассылка текстовых сообщений и медиафайлов (эффективно обрабатывается асинхронно).
- 🚫 Система управления банами с пагинацией (страничным выводом кнопок) и автоматическими уведомлениями для пользователей.
- ⚡ **Быстрая отправка:** возможность отправить личное сообщение любому пользователю, написав `<Имя> <Сообщение>` прямо в чат.

## Настройка и запуск

### 1. Конфигурация проекта
В корневой папке проекта создайте или измените файл `config.properties`:
```properties
bot.token=ВАШ_ТОКЕН_БОТА
owner.id=ВАШ_ID_ВЛАДЕЛЬЦА
bot.username=ИМЯ_ВАШЕГО_БОТА
```
*`owner.id` — это ваш уникальный номер в Telegram. Только этот человек получает доступ к кнопкам управления.*

### 2. Сборка с использованием Maven
```bash
mvn package
```

### 3. Запуск сервера
```bash
java -cp target/telegram-userbot-1.0-SNAPSHOT.jar com.example.TelegramUserBot
```

## Описание основных функций (Для Владельца)

При написании боту любого сообщения, вы активируете Панель Управления.

- **📊 Статистика:** вывод количества зарегистрированных, активных и забаненных людей.
- **📋 Список пользователей:** текстовый реестр пользователей (Имя, ID, Статус).
- **✉️ Отправить сообщение / 📤 Отправить медиа:** выбор пользователя через удобное меню-пагинацию для приватного контакта.
- **📢 Отправить всем / 🖼️ Рассылка фото:** массовая асинхронная рассылка (забаненные юзеры ее не получают).
- **🚫 Забанить / ✅ Разбанить:** блокировка доступа. Сопровождается забавными системными сообщениями.

## Технические особенности (`TelegramUserBot.java`)

1. Достигнута полная потокобезопасность (`ConcurrentHashMap`, `volatile` переменные модели данных, `AtomicBoolean`).
2. Оптимизированное потребление ресурсов. Бот использует единственный `ScheduledExecutorService` для фоновых сохранений и для пакетного планирования рассылок без блокировки основного потока (`Thread.sleep`).
3. Механика **"Умного поиска"**: администратор может не пользоваться кнопками, а просто написать `Ruslan привет!` в общий чат. Бот идентифицирует `Ruslan`, вырежет префикс имени и направит `привет!` лично получателю.
